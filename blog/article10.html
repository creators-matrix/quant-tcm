当归四逆汤与桂枝汤的基底重合：
用代码逻辑拆解方剂演变

很多人学习《伤寒论》时， 往往把每一张方剂当作“独立药方”记忆。 于是， 桂枝汤是桂枝汤， 当归四逆汤是当归四逆汤， 小建中汤又是另一套体系。 但如果真正深入研究经方结构， 就会发现： 张仲景的方剂系统， 更像一套高度模块化的“函数库”。 基础方是核心底层逻辑， 加减药味则像程序中的变量覆盖、参数继承与功能扩展。 从这个角度重新理解当归四逆汤， 可以清晰看到： 它并不是凭空创造的新方， 而是在桂枝汤基底之上， 针对“血虚寒厥”状态进行的一次结构性升级。

一、为什么经方体系更像“代码库”而不是“药方合集”？

现代人学习中医时， 最大的问题之一， 是习惯用“背方歌”的方式理解方剂。

这种方式会导致一个严重误区：

认为每一张经方， 都是彼此孤立、互不相关的。

但实际上， 《伤寒论》的方剂结构高度统一。

如果用程序员思维理解， 仲景方更像： 

BaseFormula + VariableExtension = NewFormula

也就是说： 很多复杂方剂， 实际上只是“基础函数”的二次封装。

二、桂枝汤为什么是《伤寒论》的“底层基类”？

桂枝汤原方： 

桂枝三两
芍药三两
甘草二两
生姜三两
大枣十二枚

从传统角度看， 它是“解肌调营卫”之方。

但从系统结构角度看， 桂枝汤真正重要的地方在于：

它建立了一个稳定的人体外周循环调节框架。

桂枝方向

  • 推动外周循环
  • 开放体表
  • 增强动力输出
  • 促进营卫流动

白芍方向

  • 抑制过度外散
  • 保护阴液
  • 稳定血管反应
  • 维持系统平衡

因此， 桂枝汤实际上是《伤寒论》中最核心的“循环控制基础模块”。

三、当归四逆汤如何“继承”桂枝汤？

当归四逆汤原方： 

当归三两
桂枝三两
芍药三两
细辛三两
甘草二两
通草二两
大枣二十五枚

仔细观察就会发现：

桂枝、芍药、甘草、大枣的结构完全保留。

换句话说， 当归四逆汤并不是推翻桂枝汤， 而是在其基础上： 

function 桂枝汤() {
    调营卫();
    改善外周循环();
}

function 当归四逆汤() {
    桂枝汤();
    增加养血();
    增加强温通络();
}

这正是典型的“函数复用”逻辑。

四、为什么当归与细辛是“新增变量”？

桂枝汤主要处理的是：

营卫不和 + 外周循环失调

但当归四逆汤面对的问题， 更进一步：

血虚 + 寒凝 + 四肢厥冷

因此， 原有“桂枝汤底层循环框架”已经不够。

必须额外增加两个关键变量：

新增药物 功能扩展 类似程序逻辑
当归 增强血液供应 增加资源池
细辛 深层温通寒凝 提升底层驱动力

五、为什么说经方不是“加减药”，而是“重构算法”？

现代中医教学常把加减理解为：

“症状A加某药， 症状B减某药。”

这种理解非常机械。

真正的经方演变， 更接近程序架构中的： 

BaseClass
    ↓
Inheritance
    ↓
Override
    ↓
NewBehavior

也就是说， 每一次加减， 实际上都在改变系统动力学行为。

六、当归四逆汤为什么仍然保留“桂枝—白芍”1:1结构？

最值得注意的是：

即使增加了当归与细辛， 当归四逆汤依然严格保留桂枝与白芍等量结构。

这意味着： 张仲景并没有推翻桂枝汤的底层平衡框架。

因为无论病机如何变化， 外周循环系统仍然必须维持：

“推动”与“制约”的动态平衡

七、用“模块化”理解《伤寒论》，为什么比背方更重要？

如果把《伤寒论》理解为：

一套可复用的医学算法库

那么很多复杂方剂， 就会突然变得清晰。

例如：

  • 桂枝加桂汤：增强动力输出
  • 桂枝加芍药汤：增强收敛保护
  • 小建中汤：增加中焦能量补偿
  • 当归四逆汤：增加血供与深层温通

本质上， 都是在同一个“桂枝系统”上进行功能扩展。

八、结语：真正的经方思维，是“系统继承”而不是“死记药味”

《伤寒论》的真正伟大， 并不仅仅在于留下了多少名方， 更在于它建立了一套高度模块化、可复用的医学结构体系。

当归四逆汤之所以经典， 并不是因为“多加了几味药”， 而是因为它在保留桂枝汤底层循环框架的前提下， 精准增加了养血与深层温通变量。

桂枝汤是基础函数

当归四逆汤是继承扩展

方剂演变本质是系统重构

因此， 真正理解经方， 不能停留在“背药味”的层面， 而必须进入结构学、比例学与系统动力学层面。 从程序员的角度重新阅读《伤寒论》， 或许反而更容易看见： 仲景从来不是在堆砌药物， 而是在编写一套关于人体运行逻辑的医学代码。
